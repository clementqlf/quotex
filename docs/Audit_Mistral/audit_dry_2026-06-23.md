## Duplications Critiques Identifiées

### 1. Normalisation des URI Inventaire.io

**Logique concernée** : Normalisation des URI (suppression du préfixe `wd:`, trim, lowercase)  
**Emplacements dupliqués** :
- `src/shared/api/InventaireService.ts:6-9`
- `src/entities/author/api/AuthorService.ts:15-18`

**Code dupliqué** :
```typescript
// InventaireService.ts
export const normalizeInventaireUri = (uri?: string | null): string => {
  if (!uri) return '';
  return uri.trim().toLowerCase().replace(/^wd:/, '');
};

// AuthorService.ts (méthode privée)
private normalizeInventaireUri(uri?: string | null): string {
  if (!uri) return '';
  return uri.trim().toLowerCase().replace(/^wd:/, '');
}
```

**Risque métier** : 
Si le format des URI Inventaire change (ex: nouveau préfixe, règle de normalisation différente), il faudra modifier les deux implémentations. Risque d'incohérence si une seule est mise à jour, entraînant des appels API incorrects ou des correspondances d'entités ratées entre les services de livres et d'auteurs.

**Correction recommandée** : 
Centraliser dans `src/shared/api/InventaireService.ts` (déjà exportée). Modifier `AuthorService.ts` pour importer et utiliser la fonction existante.

### 2. Calcul de backoff exponentiel

**Logique concernée** : Délai de retry avec backoff exponentiel (plafond à 1 minute)  
**Emplacements dupliqués** :
- `src/shared/lib/offline/OperationQueue.ts:149-150`
- `src/features/scanner/api/ScanService.ts:150`
- `src/entities/quote/lib/useNetworkSync.ts:150`

**Code dupliqué** :
```typescript
// OperationQueue.ts
getBackoffDelay(retryCount: number): number {
  return Math.min(1000 * Math.pow(2, retryCount), 60000); // Max 1 minute
}

// ScanService.ts (dans triggerSync)
const backoffDelay = Math.min(1000 * Math.pow(2, maxRetry), 60000); // Max 1 minute

// useNetworkSync.ts (dans triggerSync)
const backoffDelay = Math.min(1000 * Math.pow(2, maxRetry), 60000); // Max 1 minute
```

**Risque métier** : 
Si la stratégie de retry doit évoluer (ex: changer le coefficient de base, le plafond, ou la formule), il faudra modifier 3 fichiers séparément. Une divergence entre ces implémentations pourrait causer des comportements de retry incohérents entre :
- La synchronisation des quotes offline
- L'import de livres depuis le scan ISBN
- La synchronisation réseau des quotes

Cela affecterait la fiabilité globale des opérations asynchrones et offline.

**Correction recommandée** : 
Créer `src/shared/lib/offline/backoff.ts`:
```typescript
export const getExponentialBackoff = (retryCount: number, maxDelay = 60000): number =>
  Math.min(1000 * Math.pow(2, retryCount), maxDelay);
```

### 3. Parsing JSON avec fallback

**Logique concernée** : Conversion des champs potentiellement stringifiés (JSON → objet) avec gestion des erreurs  
**Emplacements dupliqués** :
- `src/entities/quote/api/SupabaseQuoteRepository.ts:50` (blockData)
- `src/entities/author/api/AuthorService.ts:23` (buyLinks)
- `src/entities/book/lib/loadBookDetailData.ts:30,97` (bookData)

**Code dupliqué** :
```typescript
// SupabaseQuoteRepository.ts
blockData: q.blockData ? (typeof q.blockData === 'string' ? JSON.parse(q.blockData) : q.blockData) : {}

// AuthorService.ts
buyLinks: b.buyLinks && typeof b.buyLinks === 'string' ? JSON.parse(b.buyLinks) : (b.buyLinks || [])

// loadBookDetailData.ts (x2)
return typeof bookData === 'string' ? JSON.parse(bookData) : bookData;
```

**Risque métier** : 
Si le format de sérialisation change (ex: ajout d'un préfixe, gestion différente des erreurs de parsing), chaque occurrence devra être mise à jour séparément. Une erreur de parsing non gérée uniformément pourrait :
- Corrompre l'affichage des données (blockData, buyLinks)
- Causer des crashes silencieux dans certains contextes mais pas d'autres
- Rendre le débogage difficile (comportement incohérent)

**Correction recommandée** : 
Ajouter à `src/shared/lib/dataHelpers.ts`:
```typescript
export const parseJsonField = <T>(value: string | T | undefined | null): T | null => {
  if (!value) return null;
  if (typeof value === 'string') {
    try { return JSON.parse(value) as T; }
    catch { return null; }
  }
  return value as T;
};
```
